## 本文幅確認用のダミーテキスト

ここから下は本文コンテンツの横幅を判断するためのダミーテキストです。確認が終わったら削除します。

技術ブログの本文というものは、短い文と長い文が入り混じって構成されるのが普通です。たとえばこの段落のように、ある程度の長さを持った説明的な文章が数行にわたって折り返されるとき、一行あたりの文字数が多すぎると視線が行末から次の行頭へ戻る際に迷子になりやすく、逆に少なすぎると視線の往復が頻繁になりすぎて、どちらの場合も読んでいて疲れる文章になってしまいます。

短い段落もあります。

一方で、技術記事には「インストールしたパッケージのバージョンとロックファイルの内容が一致していることを確認してから、開発サーバーを再起動してキャッシュを無効化する」といった、区切りの少ない長い手順説明が登場することも多く、こうした文がどのように折り返されるかも幅を判断する材料になります。カタカナの専門用語やコンポーネント、ディレクトリといった長い単語が行末に来たときの見え方も確認しておきたいところです。

コードブロックの見え方も確認します。

```tsx
export default function BlogLayout(props: Props): ReactNode {
  const {sidebar, toc: _toc, children, ...layoutProps} = props;
  return (
    <Layout {...layoutProps}>
      <div className="container margin-vert--lg">
        <div className="row">
          <main className="col col--8">{children}</main>
          <BlogSideMenu sidebar={sidebar} />
        </div>
      </div>
    </Layout>
  );
}
```

- 箇条書きの項目が一行に収まる場合の見え方
- 箇条書きの項目が長くなって折り返される場合の見え方も確認しておく必要があります。項目のインデント位置で折り返された行がどう揃うかがポイントです
- `インラインコード` を含む項目

最後に、`--ifm-container-width-xl` のようなインラインコードが本文中に登場する場合の行の高さと折り返しを確認して、ダミーテキストはおしまいです。
